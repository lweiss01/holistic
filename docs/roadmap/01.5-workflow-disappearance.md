**Priority:** High

## Goal

Make Holistic feel increasingly automatic so the user can **open repo, start working, Holistic quietly keeps continuity alive** — the core product north star.

This phase turns Holistic from a useful tool into an invisible safety net. The user should feel like continuity "just works" most of the time without thinking about commands, checkpoints, or handoffs.

### Task 1: Implicit Resume on MCP Connect

**Goal:** When an MCP client connects to `holistic serve`, automatically send the resume context without requiring an explicit `holistic_resume` tool call.

**Why This Matters:**
- **Zero-ceremony startup** - agent gets context immediately when connection opens
- **Feels like magic** - no "run holistic resume" step
- **Consistent across all MCP clients** - works the same in Claude Desktop, Cursor, etc.

**Implementation:**

1. **Add connection lifecycle hook to MCP server (`src/mcp-server.ts`):**
```typescript
import { notificationSchema } from "@modelcontextprotocol/sdk/types.js";

// After server initialization, send resume context on connect
server.onInitialize = async () => {
  const state = loadState();
  const paths = getRuntimePaths();
  
  if (state.activeSession) {
    const resumePayload = getResumePayload(state);
    
    // Send notification to client with resume context
    await server.notification({
      method: "notifications/message",
      params: {
        level: "info",
        logger: "holistic",
        data: {
          type: "resume",
          session: state.activeSession.id,
          goal: state.activeSession.currentGoal,
          status: state.activeSession.latestStatus,
          recap: resumePayload.recap,
          pendingWork: state.pendingWork.length,
        },
      },
    });
  }
};
```

### Task 2: Auto-Session Inference

**Goal:** If no active session exists, automatically start one based on recent git activity, pending work, or last handoff instead of forcing the user to run `holistic start-new`.

**Why This Matters:**
- **Removes manual session start ceremony** - work continues naturally
- **Intelligent defaults** - infers goal from context
- **Resilient to interruptions** - handles crashes, reboots, context switches

**Implementation:**

1. **Add session inference logic (`src/core/state.ts`):**
```typescript
export async function inferSessionStart(): Promise<{
  goal: string;
  plan: string[];
  source: "pending" | "handoff" | "git" | "default";
}> {
  const state = loadState();
  const git = await getGitContext();
  
  // Priority 1: Most recent pending work
  if (state.pendingWork.length > 0) {
    const top = state.pendingWork[0];
    return {
      goal: top.title,
      plan: [top.recommendedNextStep || "Continue from where we left off"],
      source: "pending",
    };
  }
  
  // Priority 2: Last handoff context
  if (state.lastHandoff) {
    return {
      goal: state.lastHandoff.title || "Continue from last handoff",
      plan: state.lastHandoff.nextSteps || ["Review last handoff and continue"],
      source: "handoff",
    };
  }
  
  // Priority 3: Recent git commits
  if (git.recentCommits.length > 0) {
    const latestCommit = git.recentCommits[0];
    return {
      goal: `Continue work on ${latestCommit.message.split('\n')[0]}`,
      plan: ["Review recent commits", "Continue current feature work"],
      source: "git",
    };
  }
  
  // Priority 4: Generic fallback
  return {
    goal: "New work session",
    plan: ["Review project state", "Start next task"],
    source: "default",
  };
}
```

2. **Update `holistic resume` to auto-start if no session (`src/cli.ts`):**
```typescript
case "resume": {

  if (!state.activeSession) {
    console.log("No active session found. Starting new session...");

    console.log(`Inferred goal from ${inferred.source}: ${inferred.goal}`);
    
    state = createSession(state, {
      title: inferred.goal,
      currentGoal: inferred.goal,
      currentPlan: inferred.plan,
      agent: argv.agent as AgentName || "unknown",
    });
    
    saveState(state);
  }
  
  // Continue with normal resume flow
  const payload = getResumePayload(state);
  console.log(formatResume(payload));
  break;
}
```

3. **Update MCP server to auto-start on connect:**

### Task 3: Smarter Passive Checkpoints

**Goal:** Create checkpoints automatically at meaningful moments (commit, branch switch, file save clusters) instead of noisy time-based polling.

**Why This Matters:**
- **Captures real work milestones** - not arbitrary 5-minute intervals
- **Lower noise** - fewer useless checkpoints
- **More useful resume context** - checkpoints match actual work rhythm

**Implementation:**

1. **Add git hook integration to auto-checkpoint on commit:**

Already implemented in Phase 1 git hook installer. Enhance it:

```bash
# .git/hooks/post-commit
#!/bin/sh
holistic checkpoint --reason "post-commit" --silent
```

2. **Add file-watch clustering for daemon (`src/daemon.ts`):**
```typescript
import chokidar from "chokidar";

let changeBuffer: string[] = [];
let checkpointTimer: NodeJS.Timeout | null = null;

const watcher = chokidar.watch(".", {
  ignored: /(^|[\/\\])\../, // ignore hidden files
  ignoreInitial: true,
  awaitWriteFinish: {
    stabilityThreshold: 2000,
    pollInterval: 100,
  },
});

watcher.on("change", (path) => {
  changeBuffer.push(path);
  
  // Debounce: checkpoint 10s after last change
  if (checkpointTimer) {
    clearTimeout(checkpointTimer);
  }
  
  checkpointTimer = setTimeout(async () => {
    if (changeBuffer.length >= 3) {
      // Meaningful cluster of changes
      await createCheckpoint({
        reason: "auto-save-cluster",
        changedFiles: changeBuffer,
      });
    }
    changeBuffer = [];
  }, 10000); // 10 second debounce
});
```

3. **Add branch-switch checkpoint (git hook):**

```bash

```

### Task 4: Auto-Drafted Handoffs

**Goal:** When idle, disconnect, or likely session end is detected, automatically draft a handoff with minimal user input required.

**Why This Matters:**
- **Captures work even when user forgets to handoff** - resilient to interruptions
- **Lower ceremony** - doesn't require explicit `holistic handoff` command
- **Better continuity** - work context preserved across unexpected disconnects

**Implementation:**

1. **Add auto-handoff detection (`src/core/state.ts`):**
```typescript
export function shouldAutoHandoff(session: SessionRecord): {
  should: boolean;
  reason: string;
} {
  const now = Date.now();
  const updated = new Date(session.updatedAt).getTime();
  const idleMinutes = (now - updated) / 60000;
  
  // Auto-handoff after 30 minutes idle
  if (idleMinutes > 30) {
    return { should: true, reason: "idle-30min" };
  }
  
  // Auto-handoff if significant work done (5+ checkpoints, 2+ hours)
  const sessionHours = (now - new Date(session.startedAt).getTime()) / 3600000;
  if (session.checkpointCount >= 5 && sessionHours >= 2) {
    return { should: true, reason: "work-milestone" };
  }
  
  return { should: false, reason: "" };
}
```

2. **Add auto-draft logic:**
```typescript

}
```

### Task 5: Automatic Holistic State Sync

**Goal:** Automatically sync `.holistic/state.json` and context docs to the `holistic/state` branch without requiring manual sync commands.

**Why This Matters:**
- **Cross-device continuity works automatically** - no manual sync needed
- **Portable memory stays up-to-date** - other devices always have latest context
- **Conservative conflict handling** - doesn't lose work

**Implementation:**

1. **Add auto-sync trigger points:**
```typescript
// In state.ts after saveState()
export async function saveStateAndSync(state: HolisticState): Promise<void> {
  saveState(state);
  
  // Auto-sync to holistic/state branch if enabled
  const config = loadConfig();
  if (config.autoSync !== false) { // default: true
    await syncStateToRemote();
  }
}
```

2. **Add conservative sync logic (`src/core/sync.ts`):**
```typescript
import { execSync } from "child_process";

export async function syncStateToRemote(): Promise<void> {
  try {
    const currentBranch = execSync("git branch --show-current").toString().trim();
    
    // Stash local state changes
    execSync("git stash push -u .holistic/", { stdio: "ignore" });
    
    // Switch to holistic/state branch
    execSync("git checkout holistic/state", { stdio: "ignore" });
    
    // Pop stashed state
    execSync("git stash pop", { stdio: "ignore" });
    
    // Check for conflicts
    const status = execSync("git status --porcelain").toString();
    
    if (status.includes("UU ")) {
      // Conflict detected - keep ours
      execSync("git checkout --ours .holistic/", { stdio: "ignore" });
      execSync("git add .holistic/", { stdio: "ignore" });
    }
    
    // Commit and push
    execSync('git commit -m "chore: auto-sync holistic state"', { stdio: "ignore" });
    execSync("git push origin holistic/state", { stdio: "ignore" });
    
    // Return to original branch
    execSync(`git checkout ${currentBranch}`, { stdio: "ignore" });
    
  } catch (err) {
    // Sync failed - log but don't crash
    console.error("Auto-sync failed:", err);
  }
}
```

3. **Add config option to disable auto-sync:**

### Task 6: One-Command Machine Bootstrap

**Goal:** Single command to install daemon, git hooks, and MCP server config on a new machine.

**Why This Matters:**
- **Onboarding is trivial** - `holistic bootstrap` and you're done
- **Cross-device setup is fast** - new laptop ready in seconds
- **Reduces manual ceremony** - no step-by-step config guide

**Implementation:**

1. **Add `holistic bootstrap` command:**
```typescript

  await installGitHooks();
  console.log("✓ Git hooks installed\n");
  
  // 2. Setup MCP server config
  console.log("Configuring MCP server...");
```

2. **Add MCP config setup:**
```typescript
async function setupMCPConfig(): Promise<void> {
  const claudeConfigPath = path.join(
    os.homedir(),
    "Library/Application Support/Claude/claude_desktop_config.json"
  );
  
  let config: any = {};
  if (await fs.access(claudeConfigPath).then(() => true).catch(() => false)) {
    config = JSON.parse(await fs.readFile(claudeConfigPath, "utf-8"));
  }
  
  config.mcpServers = config.mcpServers || {};
  config.mcpServers.holistic = {
    command: "holistic",
    args: ["serve"],
  };
  
  await fs.mkdir(path.dirname(claudeConfigPath), { recursive: true });
  await fs.writeFile(claudeConfigPath, JSON.stringify(config, null, 2));
}
```

## Regression Risks

- **Do not make auto-features too aggressive** - should feel helpful, not invasive
- **Auto-sync must handle conflicts conservatively** - never lose local work
- **Auto-sessions should not override explicit user intent** - if user wants no session, respect that
- **Auto-checkpoints should not create noise** - quality over quantity
- **Bootstrap should be idempotent** - safe to run multiple times without breaking existing config
