# Roadmap: Agent Integration Examples

**Priority:** Medium-High  
**Complexity:** Low-Medium  
**Dependencies:** Core CLI (exists), Daemon (optional for some integrations)  
**Estimated effort:** 2-3 sessions  
**Affected areas:** `adapters`, `docs`, `ux`, `architecture`

## Goal

Lower the barrier to adoption by providing concrete, copy-paste integration examples for popular tools and workflows. Move from "here's what agents should do" (adapter docs) to "here's exactly how to do it" (runnable code).

## Current State

- ✅ Adapter markdown docs exist (`.holistic/context/adapters/`)
- ✅ CLI commands work well manually
- ❌ No automated integrations for any tools
- ❌ No GitHub Actions workflow
- ❌ No VS Code extension
- ❌ No pre-commit hooks
- ❌ No shell aliases/functions documented
- ❌ No slash-command examples

## Success Criteria

1. GitHub Actions workflow for PR-based resume
2. Pre-commit hook template for auto-checkpoint
3. VS Code extension stub with workspace open integration
4. Shell helper functions for common workflows
5. Example slash-command integration for tools that support it
6. Integration guide for each major AI coding tool
7. All examples tested and documented

### Phase 1: GitHub Actions Integration (Session 1)

**Goal:** Auto-resume Holistic context when PRs are opened, providing AI agents reviewing PRs with project memory.

**Tasks:**

1. **Create workflow template `.github/workflows/holistic-pr-context.yml`:**
   ```yaml
   name: Holistic PR Context
   
   on:
     pull_request:
       types: [opened, synchronize, reopened]
   
   jobs:
     holistic-context:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
           with:
             fetch-depth: 0  # Need full history for holistic state
         
         - name: Setup Node.js
           uses: actions/setup-node@v4
           with:
             node-version: '24'
         
         - name: Install Holistic
           run: npm install -g holistic
         
         - name: Pull Holistic State
           run: |
             # Fetch state branch if it exists
             if git ls-remote --heads origin holistic/state; then
               git fetch origin holistic/state:holistic/state
               # Restore state to working tree
               .holistic/system/restore-state.sh || echo "No restore script found"
             fi
         
         - name: Generate Holistic Resume
           id: resume
           run: |
             RESUME=$(holistic resume --agent claude --json)
             echo "resume=$RESUME" >> $GITHUB_OUTPUT
             
             # Extract key context
             GOAL=$(echo $RESUME | jq -r '.activeSession.currentGoal // .lastHandoff.summary // "No active goal"')
             NEXT_STEPS=$(echo $RESUME | jq -r '.recap[] | select(. | startswith("Try next:"))' | sed 's/Try next: //')
             REGRESSIONS=$(echo $RESUME | jq -r '.recap[] | select(. | startswith("Regression watch:"))' | sed 's/Regression watch: //')
             
             echo "goal=$GOAL" >> $GITHUB_OUTPUT
             echo "next_steps=$NEXT_STEPS" >> $GITHUB_OUTPUT
             echo "regressions=$REGRESSIONS" >> $GITHUB_OUTPUT
         
         - name: Comment PR with Context
           uses: actions/github-script@v7
           with:
             script: |
               const resume = ${{ steps.resume.outputs.resume }};
               const goal = `${{ steps.resume.outputs.goal }}`;
               const nextSteps = `${{ steps.resume.outputs.next_steps }}`;
               const regressions = `${{ steps.resume.outputs.regressions }}`;
               
               const body = `## 🧠 Holistic Context
               
               **Current Objective:** ${goal}
               
               ${nextSteps ? `**Recommended Next Steps:**\n${nextSteps}\n` : ''}
               ${regressions ? `**⚠️ Regression Watch:**\n${regressions}\n` : ''}
               
               <details>
               <summary>Full Holistic Resume</summary>
               
               \`\`\`json
               ${JSON.stringify(resume, null, 2)}
               \`\`\`
               </details>
               
               *This context was automatically generated by Holistic. [Learn more](https://github.com/lweiss01/holistic)*
               `;
               
               github.rest.issues.createComment({
                 owner: context.repo.owner,
                 repo: context.repo.repo,
                 issue_number: context.issue.number,
                 body: body
               });
   ```

2. **Create PR template `.github/pull_request_template.md`:**
   ```markdown

   <!-- Describe your changes -->

   <!-- The Holistic bot will add context here automatically -->

   ```

### Phase 2: Pre-Commit Hook & Git Integration (Session 2)

**Goal:** Auto-checkpoint before every commit so no work is ever lost.

**Tasks:**

1. **Create hook installer `bin/install-git-hooks.sh`:**
   ```bash
   #!/bin/bash
   set -e
   
   REPO_ROOT=$(git rev-parse --show-toplevel)
   HOOK_DIR="$REPO_ROOT/.git/hooks"
   
   echo "Installing Holistic git hooks..."
   
   # Pre-commit: Auto-checkpoint
   cat > "$HOOK_DIR/pre-commit" << 'EOF'
   #!/bin/bash
   # Holistic auto-checkpoint before commit
   
   # Only checkpoint if holistic is initialized
   if [ -f ".holistic/state.json" ]; then
     # Get commit message from git
     BRANCH=$(git rev-parse --abbrev-ref HEAD)
     
     # Create checkpoint with commit context
     holistic checkpoint \
       --reason "pre-commit on $BRANCH" \
       --status "About to commit changes" \
       2>/dev/null || true
   fi
   
   exit 0
   EOF
   
   # Post-commit: Update changed files
   cat > "$HOOK_DIR/post-commit" << 'EOF'
   #!/bin/bash
   # Holistic: Update state after commit
   
   if [ -f ".holistic/state.json" ]; then
     COMMIT_MSG=$(git log -1 --pretty=%B)
     holistic checkpoint \
       --reason "post-commit" \
       --done "Committed: $COMMIT_MSG" \
       2>/dev/null || true
   fi
   
   exit 0
   EOF
   
   # Post-checkout: Auto-resume on branch switch
   cat > "$HOOK_DIR/post-checkout" << 'EOF'
   #!/bin/bash
   # Holistic: Resume on branch switch
   
   PREV_BRANCH=$1
   NEW_BRANCH=$2
   BRANCH_SWITCH=$3
   
   if [ "$BRANCH_SWITCH" = "1" ] && [ -f ".holistic/state.json" ]; then
     echo "🧠 Holistic: Switched to branch $(git rev-parse --abbrev-ref HEAD)"
     echo "   Run 'holistic resume' to see project context."
   fi
   
   exit 0
   EOF
   
   chmod +x "$HOOK_DIR/pre-commit"
   chmod +x "$HOOK_DIR/post-commit"
   chmod +x "$HOOK_DIR/post-checkout"
   
   echo "✓ Holistic git hooks installed"
   echo "  pre-commit: Auto-checkpoint before every commit"
   echo "  post-commit: Record commit in Holistic state"
   echo "  post-checkout: Reminder to resume on branch switch"
   ```

2. **Integrate into `holistic init`:**
   ```typescript
   // src/cli.ts
   async function handleInit(rootDir: string, parsed: ParsedArgs): Promise<number> {
     const result = initializeHolistic(rootDir, {
       // ... existing options ...

     });
     
     if (result.gitHooksInstalled) {
       process.stdout.write(`Git hooks installed: pre-commit, post-commit, post-checkout\n`);
     }
     
     return 0;
   }
   ```

3. **Add hook configuration to `.holistic/config.json`:**

### Phase 3: Shell Helpers & CLI Aliases (Session 2, continued)

**Goal:** Provide convenient shell functions for common workflows.

**Tasks:**

1. **Create shell helpers file `.holistic/system/shell-helpers.sh`:**
   ```bash
   #!/bin/bash
   # Holistic shell helpers - source this in your .bashrc or .zshrc
   
   # Quick handoff
   alias hh='holistic handoff'
   
   # Quick resume
   alias hr='holistic resume'
   
   # Quick checkpoint with current time
   alias hc='holistic checkpoint --reason "checkpoint-$(date +%H:%M)"'
   
   # Checkpoint and commit together
   hcc() {
     local message="${1:-Update}"
     holistic checkpoint --reason "pre-commit: $message" && \
     git add -A && \
     git commit -m "$message"
   }
   
   # Resume with automatic agent detection
   hra() {
     # Detect which agent based on environment
     local agent="unknown"
     if [ -n "$CURSOR_SESSION" ]; then
       agent="codex"
     elif [ -n "$CLAUDE_SESSION" ]; then
       agent="claude"
     elif [ -n "$ANTIGRAVITY_SESSION" ]; then
       agent="antigravity"
     fi
     
     holistic resume --agent "$agent" --continue
   }
   
   # Show Holistic status with git status
   hgs() {
     git status
     echo ""
     echo "🧠 Holistic Status:"
     holistic resume --json | jq -r '.recap[]'
   }
   
   # Start new session with interactive prompt
   hn() {
     read -p "Session goal: " goal
     read -p "Session title (optional): " title
     
     if [ -z "$title" ]; then
       holistic start-new --goal "$goal"
     else
       holistic start-new --goal "$goal" --title "$title"
     fi
   }
   
   # Quick sync
   alias hs='holistic sync'
   alias hp='holistic pull'
   
   # Show full context
   hctx() {
     echo "=== HOLISTIC CONTEXT ==="
     echo ""
     holistic resume
     echo ""
     echo "=== PROJECT HISTORY ==="
     head -50 .holistic/context/project-history.md
     echo ""
     echo "=== REGRESSION WATCH ==="
     head -30 .holistic/context/regression-watch.md
   }
   ```

3. **Extension code `integrations/vscode/extension.js`:**
   ```javascript
   const vscode = require('vscode');
   const { execSync } = require('child_process');
   const path = require('path');
   
   function activate(context) {
     console.log('Holistic extension activated');
     
     // Show welcome message on workspace open
     const workspacePath = vscode.workspace.workspaceFolders?.[0]?.uri.fsPath;
     if (workspacePath && hasHolisticInit(workspacePath)) {
       showResumePrompt(workspacePath);
     }
     
     // Register commands
     context.subscriptions.push(
       vscode.commands.registerCommand('holistic.resume', () => resumeSession(workspacePath)),
       vscode.commands.registerCommand('holistic.checkpoint', () => createCheckpoint(workspacePath)),
       vscode.commands.registerCommand('holistic.handoff', () => finishHandoff(workspacePath)),
       vscode.commands.registerCommand('holistic.showContext', () => showContext(workspacePath))
     );
     
     // Register tree view providers
     const contextProvider = new HolisticContextProvider(workspacePath);
     vscode.window.registerTreeDataProvider('holistic-context', contextProvider);
   }
   
   function hasHolisticInit(workspacePath) {
     const fs = require('fs');
     return fs.existsSync(path.join(workspacePath, 'HOLISTIC.md'));
   }
   
   async function showResumePrompt(workspacePath) {
     const choice = await vscode.window.showInformationMessage(
       '🧠 Holistic project detected. Resume session?',
       'Resume',
       'Start New',
       'Skip'
     );
     
     if (choice === 'Resume') {
       resumeSession(workspacePath);
     } else if (choice === 'Start New') {
       const goal = await vscode.window.showInputBox({ prompt: 'Session goal:' });
       if (goal) {
         execSync(`holistic start-new --goal "${goal}"`, { cwd: workspacePath });
         vscode.window.showInformationMessage(`✓ Started new Holistic session`);
       }
     }
   }
   
   function resumeSession(workspacePath) {
     try {
       const resume = execSync('holistic resume --json', { cwd: workspacePath, encoding: 'utf8' });
       const data = JSON.parse(resume);
       
       // Show context in output channel
       const outputChannel = vscode.window.createOutputChannel('Holistic');
       outputChannel.clear();
       outputChannel.appendLine('🧠 Holistic Context\n');
       data.recap.forEach(line => outputChannel.appendLine(`  ${line}`));
       outputChannel.show();
       
     } catch (error) {
       vscode.window.showErrorMessage(`Holistic resume failed: ${error.message}`);
     }
   }
   
   function createCheckpoint(workspacePath) {
     vscode.window.showInputBox({ prompt: 'Checkpoint reason:' }).then(reason => {
       if (reason) {
         execSync(`holistic checkpoint --reason "${reason}"`, { cwd: workspacePath });
         vscode.window.showInformationMessage(`✓ Checkpoint created: ${reason}`);
       }
     });
   }
   
   function finishHandoff(workspacePath) {
     // Show handoff form in webview
     const panel = vscode.window.createWebviewPanel(
       'holisticHandoff',
       'Holistic Handoff',
       vscode.ViewColumn.One,
       {}
     );
     
     panel.webview.html = getHandoffHtml();
     
     // Handle handoff submission
     panel.webview.onDidReceiveMessage(message => {
       if (message.command === 'submit') {
         execSync(`holistic handoff --summary "${message.summary}"`, { cwd: workspacePath });
         vscode.window.showInformationMessage('✓ Handoff complete');
         panel.dispose();
       }
     });
   }
   
   function showContext(workspacePath) {
     const fs = require('fs');
     const holisticPath = path.join(workspacePath, 'HOLISTIC.md');
     const doc = vscode.workspace.openTextDocument(holisticPath);
     vscode.window.showTextDocument(doc, { preview: false });
   }
   
   class HolisticContextProvider {
     constructor(workspacePath) {
       this.workspacePath = workspacePath;
     }
     
     getTreeItem(element) {
       return element;
     }
     
     getChildren(element) {
       if (!element) {
         // Root level - show current state
         try {
           const resume = execSync('holistic resume --json', { 
             cwd: this.workspacePath, 
             encoding: 'utf8' 
           });
           const data = JSON.parse(resume);
           
           return [
             new vscode.TreeItem(`Goal: ${data.activeSession?.currentGoal || 'None'}`),
             new vscode.TreeItem(`Status: ${data.activeSession?.latestStatus || 'No active session'}`),
             new vscode.TreeItem(`Next: ${data.activeSession?.nextSteps[0] || 'Run holistic resume'}`),
           ];
         } catch (error) {
           return [new vscode.TreeItem('Error loading context')];
         }
       }
       return [];
     }
   }
   
   function getHandoffHtml() {
     return `<!DOCTYPE html>
     <html>
     <head>
       <style>
         body { font-family: var(--vscode-font-family); padding: 20px; }
         textarea { width: 100%; height: 100px; margin: 10px 0; }
         button { padding: 10px 20px; }
       </style>
     </head>
     <body>
       <h2>Holistic Handoff</h2>
       <label>Summary:</label>
       <textarea id="summary" placeholder="What was accomplished in this session?"></textarea>
       
       <button onclick="submit()">Finish Handoff</button>
       
       <script>
         const vscode = acquireVsCodeApi();
         function submit() {
           vscode.postMessage({
             command: 'submit',
             summary: document.getElementById('summary').value
           });
         }
       </script>
     </body>
     </html>`;
   }
   
   module.exports = { activate };
   ```
